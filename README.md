# VPN Subscription Dashboard

Самостоятельный веб-сервис из двух частей:

- **Админ-панель** (для одного администратора) — управление пользователями, серверами Hiddify,
  привязками и подписками.
- **Витрина подписок** (для конечных пользователей) — по персональной ссылке `/u/{token}`
  человек видит актуальный набор subscription-ссылок с копированием и QR-кодом.

Стек: .NET 10, Blazor Server (InteractiveServer) + MudBlazor, EF Core + SQLite, Serilog, Docker.
Подробное задание — в [`VPN_DASHBOARD_PLAN.md`](VPN_DASHBOARD_PLAN.md).

## Архитектура (коротко)

- Локальная БД хранит пользователей, серверы, привязки, ручные и общие подписки. Источник правды
  по VPN-конфигам остаётся за панелями Hiddify; приватные ключи у нас **не хранятся**.
- Один локальный пользователь имеет разные UUID на разных серверах (привязка `(ServerId, Uuid)`).
- Витрина рендерится как статический SSR-компонент без Blazor-circuit — открывается быстро и
  работает даже при недоступном SignalR. QR генерируется на сервере (QRCoder).
- Админка спрятана за секретным путём `/{guid1}/{guid2}/…` (первый барьер) и закрыта
  cookie-аутентификацией с паролем (PBKDF2). Без авторизации — редирект на логин; обращение к
  админ-путям без секрета — нейтральный 404.

## Конфигурация

Все секреты задаются через переменные окружения (см. [`.env.example`](.env.example)):

| Переменная | Назначение |
|---|---|
| `Admin__SecretPathSegment1` / `…Segment2` | Два GUID-сегмента секретного пути админки |
| `Admin__PasswordHash` / `Admin__PasswordSalt` | PBKDF2-хеш и соль пароля админа (base64) |
| `ConnectionStrings__DefaultConnection` | Строка подключения SQLite (в Docker по умолчанию `/data/vpndashboard.db`) |
| `Ping__IntervalMinutes` / `…TimeoutSeconds` / `…Enabled` | Параметры фонового пинга (необязательно) |

### Сгенерировать хеш пароля

```bash
dotnet run --project VpnDashboard -- hash-password "ВАШ_ПАРОЛЬ"
# выведет строки Admin__PasswordHash=… и Admin__PasswordSalt=… — скопируйте в .env
```

В контейнере (ENTRYPOINT уже `dotnet VpnDashboard.dll`):

```bash
docker compose run --rm --no-deps vpndashboard hash-password "ВАШ_ПАРОЛЬ"
```

### Секретный путь

Сгенерируйте два GUID (`uuidgen`, онлайн-генератор и т.п.) и впишите в `Admin__SecretPathSegment1/2`.
Админка будет доступна по `https://ваш-хост/{seg1}/{seg2}/`.

## Деплой на VPS (Ubuntu 22.04) одной командой

На отдельном сервере с доменом. Сначала направьте **A-запись домена на IP этого сервера**,
затем:

```bash
git clone <repo-url> vpndashboard && cd vpndashboard
sudo bash deploy.sh
```

Скрипт сам: поставит Docker, спросит домен и пароль администратора, сгенерирует секретный путь
и хеш пароля, запишет `.env` и поднимет всё через `docker compose`. В конце он напечатает полный
адрес админки `https://домен/{seg1}/{seg2}/`.

Что разворачивается:

- **`vpndashboard`** — приложение, слушает HTTP только внутри docker-сети (порт `8080`).
- **`caddy`** — reverse-proxy на портах `80`/`443`, **автоматически** получает и продлевает
  TLS-сертификат Let's Encrypt по вашему домену. Конфиг — [`Caddyfile`](Caddyfile).
- SQLite-файл — в volume `vpndashboard-data` (`/data`), переживает пересоздание контейнера.
- Приложение доверяет `X-Forwarded-Proto`/`X-Forwarded-For` от Caddy, поэтому Secure-cookie за
  HTTPS работают корректно.

Полезное:

```bash
docker compose logs -f          # логи
docker compose pull && docker compose up -d --build   # обновление после git pull
docker compose down             # остановить
```

> Если сертификат не выдаётся — проверьте, что домен резолвится на этот сервер и порты 80/443
> открыты (firewall/провайдер). Caddy запрашивает сертификат при первом обращении к домену.

## Первые шаги после запуска

1. Откройте `https://ваш-хост/{seg1}/{seg2}/` → введите пароль администратора.
2. **Серверы** → «Добавить сервер»: домен, оба proxy-path (admin и client), admin UUID
   (Hiddify-API-Key). Кнопкой «Проверить связь» убедитесь, что панель доступна.
3. **Пользователи** → «Новый пользователь». В карточке пользователя добавьте привязки к серверам
   (к существующему UUID или создав нового через API), ручные ссылки и общие подписки.
4. Скопируйте ссылку витрины (иконка копирования в строке пользователя) и передайте человеку.

## Локальная разработка

```bash
dotnet run --project VpnDashboard --launch-profile https
```

Dev-секреты (секретный путь и пароль `devpass123`) заданы в
`VpnDashboard/appsettings.Development.json` — только для локального запуска.

## Замечания по безопасности

- Приватные ключи Hiddify (`ed25519_private_key`, `wg_pk`, `wg_psk`) не сохраняются в БД и не логируются.
- Токен витрины — криптослучайный (32 байта, base64url); кнопка «Перевыпустить токен» немедленно
  ломает старую ссылку.
- Витрина отдаёт заголовок `X-Robots-Tag: noindex, nofollow`.
- `.env` и `*.db` исключены из гита.

VPN Dashboard — это **административная панель для организации и отображения ссылок**. Это
вспомогательный инструмент управления, а не VPN-сервис.

- Программа **не предоставляет средств обхода блокировок**, не маскирует и не шифрует трафик,
  не является VPN-, proxy- или туннельным сервисом и **не проксирует пользовательский трафик**.
- Она **не генерирует и не хранит** VPN-конфигурации и приватные ключи. Сами конфигурации
  создаются и хранятся во внешних панелях (Hiddify); панель лишь сохраняет метаданные и
  отображает уже существующие subscription-ссылки, которые ввёл или к которым привязался
  администратор.
- Программа **не предоставляет доступ к каким-либо серверам** и не поставляется вместе с ними.
